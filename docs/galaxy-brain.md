# Galaxy Brain Playbook

Goal: get 2 of your GitHub Discussion answers marked as accepted.

## 1) Pick the right discussions

Prioritize:

- Repos with active Discussion traffic
- Questions with clear "solved" outcomes
- Threads with no accepted answer yet
- Recent posts (last 48-72 hours)

Avoid:

- Opinion threads
- Long architecture debates
- Posts missing enough details to solve

## 2) Use a high-acceptance answer structure

1. Restate the problem in one line.
2. Give likely root cause in one line.
3. Provide exact steps with commands.
4. Add verification steps ("you should see...").
5. Offer fallback if first fix fails.
6. End with a polite acceptance prompt.

Example closing line:

> If this fixed it, feel free to mark this as the accepted answer so others can find it quickly.

## 3) Daily execution target

- 5 thoughtful answers/day
- 60-70% should include runnable commands
- 100% should include verification steps

Expected path:

- Day 1-3: no acceptance, build quality and consistency
- Day 4-10: first accepted answer likely
- Day 10-20: second accepted answer likely

## 4) Quality checklist before posting

- Mention version/runtime (Node, Python, framework version)
- Include minimal repro assumptions
- Keep answer skimmable with bullets
- Confirm no unsafe/destructive commands
- Link official docs when possible

## 5) Where to answer first

- Popular frameworks and toolchains you already know
- Repo discussions where maintainers are active daily
- "Help / Q&A" categories with accepted answer feature enabled

# Starstruck Launch Checklist (16 Stars)

Goal: publish one public repository and get to 16 stars.

## Before launch (Day 0)

- [ ] Repo name is short, descriptive, and searchable
- [ ] README has a one-line value proposition
- [ ] Quickstart works in under 5 minutes
- [ ] Screenshots/GIF demo included
- [ ] Topics/tags added (5-10 relevant topics)
- [ ] MIT/Apache license included
- [ ] At least 2 real usage examples

## Launch week plan

### Day 1: Publish + social release

- [ ] Publish on GitHub as public
- [ ] Post launch thread on X
- [ ] Post in 1-2 relevant subreddits (respect rules)
- [ ] Post on LinkedIn and Dev.to
- [ ] Share in 2 niche Discord/Slack communities

Target: 5-8 stars.

### Day 2-3: Improve from feedback

- [ ] Fix setup friction from first users
- [ ] Add FAQ to README
- [ ] Add one feature users requested
- [ ] Reply quickly to every issue/comment

Target: 9-12 stars.

### Day 4-7: Second wave

- [ ] Ship "v0.2" release with changelog
- [ ] Post update with before/after demo
- [ ] Cross-post tutorial or short walkthrough
- [ ] Ask friends/colleagues for honest feedback (not star begging)

Target: 16+ stars.

## Star conversion boosters

- Make first screen in README visual and obvious.
- Keep install commands copy-paste ready.
- Add a "Why not X?" comparison section.
- Include badges (license, stars, last release).
- Keep examples runnable without account setup.

## What to avoid

- Generic portfolio repos with no clear utility
- Empty boilerplate without opinionated value
- Overpromising features not implemented
- Asking strangers directly to "please star"

# GitHub Achievements Playbook

Practical, ethical tactics to earn GitHub achievements faster, with a focus on:

- `Galaxy Brain` (accepted Discussion answers)
- `Starstruck` (16+ stars on one public repo)

## Why this exists

Most achievement advice is vague. This repo gives you copy-paste templates, a daily process, and a launch checklist you can actually execute.

## Quick start

1. Read `docs/galaxy-brain.md` and start with 5 targeted Discussion replies today.
2. Use `templates/discussion-reply-template.md` when you answer.
3. Follow `docs/starstruck-launch-checklist.md` to launch and promote your public repo.
4. Track weekly progress in `tracking/scoreboard.md`.

## What is inside

- `docs/galaxy-brain.md` - workflow to get accepted answers
- `docs/starstruck-launch-checklist.md` - tactical plan to reach 16 stars
- `templates/discussion-reply-template.md` - high-conversion answer template
- `templates/launch-posts.md` - launch copy for X, Reddit, and LinkedIn
- `tracking/scoreboard.md` - simple KPI tracker

## Golden rules

- Help first. Do not spam low-quality replies.
- Be specific: commands, versions, and repro steps.
- Ask for acceptance politely, never aggressively.
- Build in public and ship updates weekly.

## Suggested weekly cadence

- Mon/Tue: 8-12 Discussion answers across active communities.
- Wed: ship one repo improvement and post an update.
- Thu/Fri: answer comments/issues and gather feedback.
- Weekend: docs polish and examples.
